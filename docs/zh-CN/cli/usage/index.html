CLI用法

此使用文档假定了你已经遵循cli installation steps。

贴士：以下我们将展示all-contributors来保持简单，但是如果你有任何困难，请尝试yarn all-contributors。

命令

<autogenerated_table_of_contents>

all-contributors init

如果第一次使用all-contributors-cli的话，请使用运行此命令。 此命令会问一些问题，同时为all-contributors设置项目。 此设置包含了创建.all-contributorsrc配置文件和在规定的files中创建贡献者列表 (通常在你的README.md)。

all-contributors add

使用add在你的项目中添加新的贡献者，或者添加新的贡献方式。 它们会被添加到你的配置文件，同时如果你使用generate command，贡献者文件将被更新。

# 添加新的贡献者 <用户名>, 和贡献类型 <贡献>
all-contributors add <username> <contribution>
# 范例：
all-contributors add jfmengels code,doc

用户名是用户GitHub或者GitLab的用户名，贡献是一个以,分隔的贡献列表。 通过Emoji Key (贡献类型引用)查看一系列有效的贡献类型。

GitLab用户：请查看GitLab用户额外规定。

all-contributors check

使用check比较GitHub贡献者和在你的.all-contributorsrc file里的贡献者，确保赞扬给在适当的地方。

因为GitHub API限制，此命令近适用于少于500名贡献者的项目。 (除非你设置了PRIVATE_TOKEN) 在below in GitHub users有提及。

all-contributors generate

Generate将从你的.all-contributorsrc文件中读取贡献者列表，并且在规定的files中更新贡献者表格。

请注意，以便更新表格，命令必须能够在这些文件中找到如下标记。

GitHub用户

在某些情况下，你可能会看到错误消息GitHub API rate limit exceeded for xxx。 你可能需要设置一个名为PRIVATE_TOKEN的环境变量，确保规避这个GitHub rate limit。

Private token是你使用GitHub API进行身份验证的私人访问令牌。

GitLab用户

请注意，如果你在使用自己搭建的gitlab实例，一些命令将需要你首先设置一个名为PRIVATE_TOKEN的环境变量。

Private token是你使用GitLab API进行身份验证的私人访问令牌。

# 在linux上设置private token
export PRIVATE_TOKEN=your_private_token
# 在windows上设置private token
set PRIVATE_TOKEN=your_private_token
